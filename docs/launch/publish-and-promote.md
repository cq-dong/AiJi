# AiJi · 上架与宣传方案

> 2026-07-17
> 目标：四端上架（鸿蒙 AppGallery + PWA + iOS + 安卓）+ 小预算宣传，主攻高频记想法的个人

---

## 0. 核心定位（所有渠道统一口径）

**一句话**：本地优先的 AI 随手记——说/打字/拍任选，AI 自动涌现分类与聚合，数据留在你设备上。

**三个差异化卖点（竞品打不动的）**：
1. **本地优先 + BYOK**：原始数据不上云，AI 用你自己的 key——隐私和成本都你说了算
2. **涌现分类**：不预定「生活/想法/项目」大类，LLM 从内容发现类别——笔记不会变成垃圾堆
3. **与库对话**：能问 AI「我上个月那个关于 X 的想法」——把笔记库变成可对话的知识库

**反竞品对照**（宣传素材隐含，不直接点名）：
- 比语音备忘：多了 AI 整理 + 多模态
- 比笔记 App（Notion/语雀）：多了自动分类 + 隐私本地
- 比日记：多了非生活类（项目/想法）的开放结构
- 比 flomo：多了多模态 + AI 对话检索

## 1. 四端上架路径

### 1.1 总策略：一套核心，多端壳

AiJi 的端口/适配器架构（CLAUDE.md §2）天然支持多端：
- **Domain + Port + 处理管线**：所有端共用（100% 复用）
- **UI 层**：鸿蒙用 ArkTS/ArkUI 重写；iOS/安卓/PWA 用 React（现有）
- **适配器**：鸿蒙换鸿蒙能力；iOS/安卓用 Capacitor 壳包现有 PWA

```
                ┌─ 鸿蒙原生（ArkTS）→ AppGallery
                │
AiJi 核心 ──────┼─ React PWA ──→ 网页可安装（PWA）
(Domain/Port)   │
                └─ React PWA + Capacitor 壳 ─→ iOS App Store
                                          └→ 安卓应用商店
```

### 1.2 鸿蒙 AppGallery（优先，配合大赛）
- **为什么优先**：C4 大赛复赛须交 HAP，顺理成章上架；鸿蒙生态应用增长期，小应用更易被推荐
- **要求**：HarmonyOS 原生（ArkTS/ArkUI），HAP 包，华为开发者账号（个人 $0）
- **加分**：集成鸿蒙特性（端侧 AI / 分布式数据 / 元服务卡片）——大赛和上架双重加分
- **时间**：随复赛进度（9 月底前出可演示 HAP，审核约 1-2 周）

### 1.3 PWA 可安装（最快，零成本起量）
- **现状**：已有 vite-plugin-pwa，但 SW 未注册（审计 B2）、manifest 冲突（A6）、无 apple-touch-icon（S4）——**先修这些**
- **部署**：任意静态托管（Vercel/Cloudflare Pages/GitHub Pages，免费）
- **优势**：零审核、零费用、即时更新、所有平台通吃
- **劣势**：发现性差（无商店流量），靠宣传导流
- **时间**：修好 PWA bug 后当天可上线

### 1.4 iOS App Store（高门槛，后期）
- **路径**：React PWA + Capacitor 壳包成原生 iOS 应用
- **要求**：苹果开发者账号 $99/年、Mac + Xcode、过苹果审核（严）
- **审核风险点**：
  - BYOK：苹果可能要求「不强制登录第三方账号」——BYOK 是用户自愿配 key，审核说明讲清「核心功能不依赖任何账号」
  - 数据采集声明：麦克风/摄像头/位置权限要解释清楚
  - 内购：若以后加云同步订阅，须走苹果内购（30% 抽成）；MVP 不加则无此问题
- **时间**：鸿蒙+PWA 跑通后再做，3-4 周开发 + 1-2 周审核

### 1.5 安卓应用商店（多渠道）
- **路径**：React PWA + Capacitor 壳包成 APK/AAB
- **渠道**：国内多商店（华为/小米/OPPO/vivo/应用宝/酷安）——**每个单独上架**
- **要求**：软著（软件著作权，国内商店必需，约 ¥300-700 代办，1 个月）、各商店开发者账号（多数免费）
- **建议**：先上华为（与鸿蒙 AppGallery 同账号体系）+ 酷安（开发者友好），再铺其他

### 1.6 上架节奏

| 阶段 | 渠道 | 时机 | 前置 |
|------|------|------|------|
| 第 1 步 | PWA 可安装 | 立刻（修好 PWA bug） | 修 B2/A6/S4 |
| 第 2 步 | 鸿蒙 AppGallery | 9 月（随大赛复赛） | 鸿蒙化 HAP |
| 第 3 步 | 安卓（华为+酷安） | 10 月 | 软著 + Capacitor 壳 |
| 第 4 步 | iOS App Store | 11 月+ | $99 账号 + 审核 |

## 3. 隐私政策要点（必须写，硬要求）

AiJi 的隐私政策有**特殊优势**——本地优先 + BYOK，可以写得非常强势：

1. **数据存储**：所有记录存储在你设备的本地存储（IndexedDB/OPFS 或鸿蒙分布式数据），**不上传到我们的服务器**——因为我们没有服务器
2. **AI 处理**：AI 分类/摘要/检索调用你配置的 LLM 服务（BYOK）。相关条目内容发送到**你所选的 LLM 提供商**（如 DeepSeek/OpenAI），受该提供商隐私政策约束。我们不经手、不存储、不中转
3. **API Key**：你的 key 存储在设备本地，不上传给我们
4. **权限**：麦克风/摄像头/位置仅在你主动使用时启用，不后台采集
5. **数据导出**：可随时导出全部数据（.zip/.md），可彻底删除

**写法**：用 pm-toolkit 的 privacy-policy skill 起草，突出「无服务器、无账号、数据归你」——这本身是营销卖点。

## 4. 宣传策略（小预算）

### 4.1 宣传节奏

| 阶段 | 目标 | 动作 |
|------|------|------|
| 预热期（上架前） | 积累种子用户 | 内容营销 + 社区 |
| 上架期（PWA+鸿蒙） | 首批下载 | 商店优化 + 发布公告 |
| 增长期（四端齐） | 口碑扩散 | KOL + 推荐 |

### 4.2 内容营销（零成本，主力）

**平台**：少数派、即刻、V2EX、小红书、B站、知乎、掘金

**选题**（每篇一个引流入口）：
1. **「为什么我做了个不联网的 AI 笔记」**——本地优先 + BYOK 隐私故事（少数派/即刻爆款向）
2. **「AI 帮你整理笔记是什么体验」**——涌现分类 + 对话检索 demo 视频（B站/小红书）
3. **「鸿蒙原生 AI 应用开发实录」**——技术向，蹭鸿蒙热度（掘金/知乎，大赛素材）
4. **「用 BYOK 把 AI 笔记月成本压到 $2」**——成本对比（V2EX/少数派）
5. **「从 React PWA 到鸿蒙原生：端口架构的胜利」**——技术深度（掘金）

**节奏**：上架前每周 1-2 篇，持续 4-6 周积累。每篇文末放 PWA 安装链接（最快可达）。

### 4.3 社区运营（零成本）
- **即刻**：AI/效率工具圈子，发产品动态 + 开发故事，圈层精准
- **V2EX**：分享创造节点，开发者用户多，愿试独立产品
- **少数派**：效率工具专栏投稿，质量高流量精准
- **酷安**：上架后求「酷安小编」推荐，独立应用友好
- **鸿蒙社区**：花粉俱乐部/鸿蒙开发者社区，蹭大赛 + 生态

### 4.4 小预算投放（几百~几千）
- **AppGallery 推荐位**：CPT/CPD，配合大赛鸿蒙热度期
- **小红书信息流**：定向「笔记/效率/学习」兴趣，demo 视频素材，¥300-700/天试跑
- **小 KOL 寄送**：5-10 个效率/学习类小 KOL（粉丝 1k-1w），免费用 + 真实评测，比大 KOL 性比高
- **酷安/少数派 Banner**：短周期投放，精准

**预算分配建议**（假设总 ¥3000）：
- AppGallery 推荐：¥1200（鸿蒙热度期）
- 小红书信息流：¥900
- 小 KOL 寄送：¥900

### 4.5 增长黑客（零成本杠杆）
- **PWA 即开即用**：宣传链接点开就能用，无需下载——转化率远高于商店。所有内容导流 PWA，满意了再引导装原生
- **导出即传播**：导出的 .md/.zip 自带「由 AiJi 生成」水印——用户分享笔记即传播
- **BYOK 自传播**：用户配 key 时会搜「DeepSeek 怎么用」——在 LLM 社区露出
- **大赛背书**：C4 获奖/晋级本身是宣传素材（「获奖鸿蒙应用」）
