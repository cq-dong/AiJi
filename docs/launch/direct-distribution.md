# AiJi · 网络下载分发方案（起步首选）

> 2026-07-17
> 定位：跳过商店上架，直接网络分发。契合 AiJi「本地优先 + BYOK」零后端起步。
> 商店上架降为远期选项（见 §7），待用户量起来/商业化时再补。

---

## 0. 为什么走网络下载

AiJi 游客模式 = 纯本地 + 自带 key = **零后端**。这个形态天然适合网络分发：
- **不需要软著**（安卓商店才要）
- **不需要 ICP 备案**（有后端才要）
- **不需要审核**（商店才要）
- **不需要抽成**（商店内购才要）
- **即时更新**（不用过审）

代价：发现性差（靠自己引流）、用户信任低（未知来源）、iOS 受限。但起步阶段这些可接受——先验证产品，再上商店。

## 1. 三种分发形态

| 形态 | 覆盖平台 | 摩擦 | 适用 |
|------|---------|------|------|
| **PWA 可安装** | 全平台（含 iOS Safari） | 最低（点链接即用） | 主入口，所有宣传导流 |
| **安卓 APK 下载** | 安卓 | 中（手动允许未知来源） | 要原生体验的用户 |
| **鸿蒙 HAP 下载** | 鸿蒙 | 中（侧载） | 配合大赛 |
| iOS 直接下载 | ❌ 不可能 | — | iOS 只能 PWA 或 TestFlight |

### 1.1 PWA 为主入口（核心）
- 修好 3 个 bug（B2 SW注册 / A6 manifest冲突 / S4 apple-touch-icon）+ 部署 HTTPS
- 用户访问网址即用，满意后「添加到主屏」变原生体验
- 零摩擦、零审核、所有平台通吃
- iOS Safari 也能加到主屏（虽无安装提示，UI 引导即可）
- **所有内容营销导流 PWA 链接**——转化率最高

### 1.2 安卓 APK 下载（补充）
- React PWA + Capacitor 壳 → 打包 APK
- 放下载页，用户手动下载 + 允许「安装未知来源」
- 优势：原生后台运行、更好的麦克风/摄像头权限、脱离浏览器
- 摩擦：手动开权限，有风险提示

### 1.3 鸿蒙 HAP 下载（配合大赛）
- C4 大赛复赛要 HAP，顺带提供下载
- 鸿蒙侧载用户少，主要给大赛评审 + 鸿蒙早期用户

### 1.4 iOS
- 不直接下载（苹果生态封闭）
- 走 PWA（iOS Safari 添加到主屏）即可覆盖大部分需求
- 若要原生 iOS：TestFlight（最多1万人，需审核）或后期上 App Store

## 2. 下载页设计

### 2.1 页面结构
```
AiJi · AI 记
本地优先的 AI 随手记

[立即体验 / 安装]  ← PWA 主按钮（点开即用）
[下载安卓 APK]     ← 次要按钮
[下载鸿蒙 HAP]     ← 次要按钮（大赛后）

立即体验无需下载，数据留在你设备上

—— 三卖点 ——
• 本地优先 + BYOK：数据不上云，AI 用你自己的 key
• 涌现分类：LLM 从内容发现类别，不预定大类
• 与库对话：问 AI「我上个月那个想法」，带引用回答

—— 信任 ——
SHA256: xxxxxxxxxxxx  ← APK 校验值
开源 / 数据可导出 / 无服务器
```

### 2.2 托管
- **PWA + 下载页**：Vercel / Cloudflare Pages / GitHub Pages（免费、HTTPS 自动）
- **APK/HAP 文件**：GitHub Releases（免费、有 CDN）或同托管站
- **域名**：自有域名最佳（提升信任），如 aiji.app；没有就用 vercel.app 子域起步

### 2.3 信任建设（网络下载关键）
网络下载最大障碍是用户怕病毒。应对：
- [ ] 提供 APK SHA256 校验值，用户可验证完整性
- [ ] 自有域名托管（别用不知名网盘）
- [ ] 网站专业（品牌素材已有）
- [ ] 明示「开源 / 无服务器 / 数据可导出」
- [ ] （可选）代码开源 GitHub，用户可自查
- [ ] （可选）APK 数字签名

## 3. 更新机制（商店自动更新没了，要自己做）

### 3.1 PWA 更新
- vite-plugin-pwa `registerType:'autoUpdate'` 自动检查新版本
- 用户刷新即更新，无需操作
- 最省心

### 3.2 APK 更新
- App 内加「检查更新」：请求版本接口 → 比对 → 提示下载新 APK
- 版本接口：一个静态 JSON（放托管站）
  ```json
  { "version": "0.2.0", "url": "https://aij.app/aiji-0.2.0.apk", "sha256": "xxx", "note": "修复..." }
  ```
- App 启动时静默检查，有新版提示用户下载安装

### 3.3 HAP 更新
- 同 APK 思路，或引导用户回下载页取新版

## 4. 引流策略（网络分发无商店流量，全靠自己）

### 4.1 内容营销（主力，零成本）
平台：少数派、即刻、V2EX、小红书、B站、知乎、掘金

选题（每篇引流到 PWA）：
1. 「为什么我做了个不联网的 AI 笔记」——本地优先+BYOK 隐私故事（少数派/即刻）
2. 「AI 帮你整理笔记是什么体验」——涌现分类+对话检索 demo（B站/小红书）
3. 「鸿蒙原生 AI 应用开发实录」——蹭鸿蒙热度（掘金/知乎，大赛素材）
4. 「用 BYOK 把 AI 笔记月成本压到 $2」——成本对比（V2EX）
5. 「从 React PWA 到鸿蒙原生」——技术深度（掘金）

节奏：每周 1-2 篇，持续 4-6 周。文末放 PWA 链接。

### 4.2 社区（零成本）
- 即刻 AI/效率圈、V2EX 分享创造、少数派专栏、酷安（后期上架后）、鸿蒙社区
- 蹭 C4 大赛背书（「获奖鸿蒙应用」是素材）

### 4.3 小预算投放（可选）
- 小红书信息流定向「笔记/效率」
- 小 KOL 寄送（5-10 个效率/学习类）
- 酷安/少数派 Banner

### 4.4 增长杠杆
- **PWA 即开即用**：宣传链接点开就能用，转化率远高于下载
- **导出带水印**：导出的 .md/.zip 自带「由 AiJi 生成」，用户分享即传播
- **BYOK 自传播**：用户配 key 时搜「DeepSeek 怎么用」，LLM 社区露出

## 5. 月活统计（无商店统计，要自己做）

网络分发没有商店的安装/活跃统计。AiJi 本地优先无后端，月活靠：
- **匿名遥测**（默认关闭，用户同意才开）：首次启动弹「是否允许发送匿名使用统计？」，同意的回传 `{anonId, date}` 去重计月活
- **PWA 访问统计**：托管站可加匿名访问统计（Cloudflare Analytics / Vercel Analytics，免费、隐私友好）
- **下载量统计**：GitHub Releases 下载次数 / 托管站 APK 下载计数
- 对外话术：「N 名用户主动分享使用数据 + M 次下载」——诚实

绝不为月活破「无服务器」承诺。统计通道 ≠ 数据通道。

## 7. 何时转上架商店

网络下载起步，**出现以下信号时**考虑上架：
- 用户量到一定规模（如 1000+ 主动用户）——商店发现性开始值
- 要做内购商业化（商店支付更可信）
- 用户反复问「在哪下载」（商店更方便）
- 大赛要求上架加分

上架时再补：软著（6周，提前办）、ICP备案（做后端才要）、各店账号。网络下载和上架可并存——PWA 永远是最低摩擦入口。

## 8. 风险与注意

| 风险 | 应对 |
|------|------|
| 用户怕 APK 是病毒 | SHA256 校验 + 自有域名 + 开源 + 专业网站 |
| 发现性差 | 内容营销持续 + PWA 即开即用降摩擦 |
| 无自动更新 | PWA autoUpdate + APK 检查更新机制 |
| 国内监管灰色 | 纯本地零后端最安全；做后端后再备案 |
| iOS 用户流失 | PWA 覆盖 iOS Safari，或后期 TestFlight |
| 月活不可信 | 匿名遥测下限 + 下载量 + 访问量组合 |
