# AiJi · 应用商店上架研究

> 2026-07-17
> 信息来源：Apple 官方文档、web.dev（PWA）、C4 大赛规程 PDF。华为/安卓部分基于公开要求 + 规程交叉验证，标⚠️待核实的点需上官方平台确认。
> 注意：本研究环境的 Web 检索受限（WebSearch 区域返空、部分站点 403/被拦），华为/小米/OPPO/vivo 官方页未能直接抓取，建议上各开放平台核实⚠️项。

## 0. 总览：四端上架对比

| 维度 | PWA 可安装 | 鸿蒙 AppGallery | 安卓多商店 | iOS App Store |
|------|-----------|----------------|-----------|---------------|
| 费用 | 免费 | 个人 $0 | 个人多数免费 | $99/年 |
| 审核 | 无 | 1-2 周 | 3-7 天 | 24-48h |
| 软著 | ❌ | ⚠️建议有 | ✅必需（国内） | ❌ |
| ICP备案 | ❌ | ⚠️付费需 | ✅必需（国内） | ✅中国区必需 |

| 建议时机 | 立刻 | 9月（随大赛） | 10月 | 11月+ |

## 1. PWA 可安装（最快，零成本）

### 1.1 硬性安装条件（Chrome/Edge）

Manifest 必须含：
- `short_name` 或 `name`
- `icons`：**必须含 192px 和 512px**
- `start_url`
- `display`：fullscreen/standalone/minimal-ui/window-controls-overlay 之一
- `prefer_related_applications`：不存在或为 false

其他：
- **HTTPS**（必须）
- 用户参与度：至少点过一次 + 浏览≥30秒（浏览器才弹安装提示）
- Service Worker：web.dev 当前安装条件**未明确要求 SW**，但建议注册以支持离线 + 满足部分浏览器

### 1.2 AiJi 现状缺口（审计已知）
- ❌ B2：SW 未注册（main.tsx 无 virtual:pwa-register）
- ❌ A6：manifest 冲突（静态 webmanifest 只列 favicon.svg vs VitePWA 列 PNG）
- ❌ S4：无 apple-touch-icon（iOS 主屏图标退化）
- ⚠️ 图标是占位（192=593B, 512=2.2KB）

**修法**：main.tsx 加 `registerSW()`；删静态 manifest 让 VitePWA 接管；加 apple-touch-icon；meigen 出真图标。

### 1.3 iOS Safari 安装差异
- iOS 不自动弹安装提示，需用户手动「分享→添加到主屏」
- 要在 UI 里引导（beforeinstallprompt 在 iOS 不触发，需检测 iOS Safari 显示自定义提示）
- apple-touch-icon 必须有（180×180 最佳）

### 1.4 PWA 上架商店
- **PWABuilder**：微软工具，可把 PWA 打包成 APK/AAB 上 Google Play、MS Store
- 不能上 App Store（苹果不接受纯 PWA 包装，要原生壳 Capacitor）

## 2. 鸿蒙 AppGallery（配合大赛）

### 2.1 为什么优先
- C4 大赛复赛须交 HAP，顺理成章上架
- 鸿蒙生态应用增长期，小应用易被推荐
- 华为开发者账号个人免费

### 2.2 上架要求（⚠️部分待官方核实）
- 华为开发者账号：个人免费，需实名认证（身份证+人脸）⚠️
- HarmonyOS 原生应用：ArkTS/ArkUI 开发，提交 HAP 包
- 应用材料：图标、截图、描述、隐私政策
- 软著：⚠️鸿蒙应用上架是否强制软著待核实（安卓应用国内必软著，鸿蒙可能同样要求）
- ICP备案：⚠️若应用有后端域名（如内置key代理、云备份），需 ICP 备案；纯本地无后端可能不需要
- 审核：1-2 周

### 2.3 鸿蒙 vs 安卓上架
- 鸿蒙原生（HAP）：ArkTS 开发，走 HarmonyOS 应用分类
- 安卓应用（APK/AAB）：常规安卓包，走 Android 应用分类
- AiJi 两个都上：鸿蒙原生版（大赛+鸿蒙生态）+ 安卓版（Capacitor 壳，覆盖老鸿蒙/其他安卓）

### 2.4 内购
- 华为应用内支付（IAP），个人开发者可接入⚠️
- 抽成比例约 30%（具体待核实）

## 3. 安卓多商店（国内）

### 3.1 渠道清单
华为、小米、OPPO、vivo、应用宝、酷安——**每个单独上架**。

### 3.2 通用要求（⚠️各店细节待核实）
- 个人开发者账号：多数免费，需实名（身份证+人脸）
- **软件著作权：国内安卓商店必需**（软著登记，约 ¥300-700 代办，1 个月）
- **ICP备案：有后端域名则必需**（无后端纯本地可能不需要，但内置key代理/云备份一定有后端）
- 应用材料：图标、截图、描述、隐私政策、权限说明
- 审核：3-7 天

### 3.3 上架顺序建议
1. **华为**（与 AppGallery 同账号体系，复用）
2. **酷安**（开发者友好，独立应用易获推荐）
3. **小米/OPPO/vivo/应用宝**（用户基数大，但维护成本高，后期铺）

### 3.4 软著办理
- 提前 1 个月办（别等要上架才申请）
- 材料：源代码（前30后30页）、说明书、申请表
- 可找代办（¥300-700，省事）

## 4. iOS App Store（高门槛）

### 4.1 账号
- **Apple Developer Program：$99/年（约 ¥688）**
- 注册要求：Apple ID + 双重认证 + 付款方式 + **政府证件验证**（Apple Developer App 人脸+证件）
- 中国区：中国大陆 Apple ID，支持支付宝/微信付年费
- 个人不需要邓白氏编码（企业才要）

### 4.2 提交材料
- 截图：iPhone 6.9寸（1290×2796）必需，最多10张
- App图标：1024×1024
- 隐私政策：**必须提供 URL**（托管自己服务器）
- 隐私标签：声明数据收集类型/用途
- 权限说明：Info.plist 每个权限的中文用途说明
- 元数据：名称(30字)、描述(4000字)、关键词(100字)

### 4.3 审核周期与拒因
- 首次：24-48h（最长7天）
- 常见拒因：崩溃Bug(2.1)、功能不完整(2.1)、Web View包装无原生功能(4.2)、隐私政策不合规(5.1.1)、权限说明不清(5.1.1)

### 4.4 BYOK 类应用审核要点（AiJi 关键）
1. **API Key 必须本地存储使用，不得上传开发者服务器**——否则视为「收集敏感信息」拒审(5.1.2)
   - ⚠️**AiJi 冲突点**：你设计的「内置key代理」会把用户内容经你的代理转发到 LLM——这**可能触发 5.1.2**。需在隐私政策明确「代理不存储内容，只转发」，且内置key是可选的（用户可全程自带key不上传任何内容）
2. 必须有完整隐私政策，说明 key 如何存储/使用/不离开设备
3. 功能完整性(4.2)：不能只是 API 调用包装，要有额外功能价值（AiJi 的多模态+分类+聚合满足）
4. 需登录的 App 必须提供测试账号给审核员

### 4.5 第三方登录规则（4.8）
- 若提供第三方登录（微信/Google 等），**必须同时提供「通过 Apple 登录」**
- AiJi 若自建账号系统（邮箱/手机注册），**不触发**此规则（仅第三方登录才要求 Apple 登录）
- ⚠️建议：AiJi 自建邮箱/手机注册即可，不接第三方登录，避开 Apple 登录要求

### 4.6 内购抽成
- 标准：30%（年收入>$100万）
- 小型企业：15%（年收入≤$100万，需加入 Small Business Program）
- 订阅超1年：降至15%
- ⚠️**AiJi 付费权益包在 iOS 必须走 IAP**（苹果抽 15-30%）。要么 iOS 涨价覆盖，要么 iOS 仅免费

### 4.7 中国区特殊要求（2023.9 起强制）
- **ICP备案：中国区上架必需**（个人需身份证+手机号+服务器域名已备案+域名所有者与开发者一致）
- 无 ICP 备案无法上架中国区 App Store
- 数据本地化：中国用户数据存境内服务器
- 首次启动隐私政策同意弹窗（不能默认勾选）
- 提供注销账号功能

## 5. AiJi 上架合规清单（综合）

### 5.1 通用
- [ ] 隐私政策（强化版，见下）
- [ ] 用户协议（含付费条款、退款）
- [ ] 应用图标 1024×1024 高清
- [ ] 商店截图 5-8 张
- [ ] 应用描述文案

### 5.2 国内（华为/安卓/iOS中国区）
- [ ] **ICP备案**（有后端域名则必需，~2周）——AiJi 内置key代理/云备份需要
- [ ] **软件著作权**（安卓必需，~1月）
- [ ] 首次启动隐私同意弹窗（不默认勾选）
- [ ] 注销账号功能
- [ ] 数据境内存储（若用国内云函数则满足）

### 5.3 iOS 专属
- [ ] Apple Developer $99/年
- [ ] Info.plist 权限说明字符串
- [ ] 隐私标签
- [ ] ICP备案（中国区）
- [ ] 测试账号（若登录）
- [ ] IAP 接入（若付费）

### 5.4 隐私政策强化要点（相比纯本地版）
1. 游客：纯本地，无服务器
2. 注册用户：账号信息（邮箱/手机）用于登录权益
3. 内置key：条目内容经代理转发到 LLM 提供商，**代理不存储内容**（只记额度）——⚠️iOS 5.1.2 风险，需明确
4. 云备份：付费用户上传加密备份，可应要求删除
5. 自带key：不经手，同纯本地
6. 匿名遥测：游客可选，只传匿名ID+日期

## 6. AiJi 上架路径决策

### 6.1 推荐顺序与时机
| 步骤 | 渠道 | 时机 | 关键前置 |
|------|------|------|---------|
| 1 | PWA | 立刻 | 修 B2/A6/S4 + 部署 HTTPS |
| 2 | 鸿蒙 AppGallery | 9月 | 鸿蒙化 HAP + 实名 |
| 3 | 安卓（华为+酷安） | 10月 | Capacitor壳 + 软著 + ICP |
| 4 | iOS | 11月+ | $99 + ICP + 审核 |

### 6.2 关键决策点
1. **内置key代理是否上 iOS**：有 5.1.2 风险。建议 iOS 版仅支持自带key（纯本地），内置key仅安卓/鸿蒙——避开审核风险
2. **iOS 付费策略**：走 IAP（抽15-30%）或 iOS 仅免费
3. **后端域名 ICP 备案**：一旦做账号/内置key/云备份就要备案，提前办
4. **软著**：安卓必需，提前1月
