# AiJi（AI 记）· 作品介绍文档

> 2026 鸿蒙高校创新赛 · 应用创新方向

## 一句话创意

AiJi 是一款本地优先的 AI 随手记应用：多模态捕获想法，LLM 涌现分类与聚合，并与自己的「记」库对话检索——数据留在端侧，AI 能力由用户自带密钥驱动。

## 创意背景

人每天会冒出大量想法、进展、生活片段，但多数转瞬即逝——不是没工具记，而是记的摩擦太高、记完没人整理、积成垃圾堆。现有工具各有短板：语音备忘只单一模态且不整理；笔记 App 多模态弱、分类要手动；日记类结构僵化只覆盖生活；云笔记数据上云不可控。

我们想要的工具是：随手记（说/打字/拍任选）→ AI 自动整理（分类、摘要、提醒）→ 日后能对话式找回 → 数据留在本地。AiJi 即「AI 记」，定位是通用的「记」，不是日记——条目异构，类别由内容涌现、不预定。

## 核心功能

1. **多模态混采**：一条记录可混合文本、语音、拍照、视频。语音实时转写预览，保存后云端 STT 出终稿。
2. **LLM 涌现分类**：不预设「生活/想法/项目」等固定大类，LLM 从内容发现类别并复用既有标签，用户可策展（合并/重命名/新增）。
3. **时间摘要聚合**：按日/周/月生成 LLM 聚合摘要，5 级详细度可调，跨条目提炼脉络。
4. **提醒意图检测**：LLM 识别「明天下午3点提醒我…」类时间意图，建议提醒供用户确认。
5. **与库对话检索**（创新亮点）：用自然语言问「我上个月关于记忆外包的那个想法」，AI 检索库内容并带引用回答——把笔记库变成可对话的知识库。
6. **本地优先 + BYOK**：原始数据存端侧（IndexedDB/OPFS），AI 调用走用户自配的 LLM key，不经手用户数据、成本可控。

## 技术实现路径

架构采用端口/适配器分层：Domain 与 Port 接口零平台耦合，采集/存储/LLM/STT 各为可替换适配器。这一设计为鸿蒙化预留退路——移植时 UI 层用 ArkTS/ArkUI 重写，Domain 与处理管线直接复用，适配器换成鸿蒙能力。

鸿蒙化将集成：① HiAI Foundation 端侧 LLM，让分类/摘要可在端侧完成，强化隐私与离线能力；② 分布式数据管理，实现手机记、平板/PC 看的多端协同；③ 原子化服务卡片，桌面一键记一笔，契合「随手记」低摩擦定位。AI 对话检索与涌现分类构成应用的技术创新亮点。

## 市场前景

目标用户为高频捕获想法与进展的个人——做项目的人、爱记灵感的人、记录生活者。隐私敏感与不愿被云锁定是强需求，本地优先 + BYOK 直击痛点。鸿蒙生态正处于应用增长期，「小而美 + AI 融合 + 实用场景」的应用契合赛事导向。AiJi 的涌现分类与对话检索区别于传统笔记固定分类与关键词搜索，具备差异化竞争力，可上架面向鸿蒙用户。
