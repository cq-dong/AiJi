## 6. 鸿蒙化技术路径（复赛，8-9月）

AiJi 架构（端口/适配器分层）已被**三次真实验证**——同一套 UI/Domain，采集/存储/AI 适配器在 PWA、Capacitor Android、服务端代理三种形态下跑通（v2.0.1 现状：可安装 APK + PWA + 自有生产后端）。鸿蒙化是同一打法再来一次：

| 层 | 现状（v2.0.1） | 鸿蒙化 | 复用度 |
|----|----------------|--------|--------|
| Domain（types/规则） | 纯 TS 零 I/O | 直接复用 | 100% |
| Port（接口） | 5 端口抽象 | 接口复用 | 100% |
| 适配器 | Dexie/Paraformer/Capacitor/服务端代理 | 换鸿蒙适配（分布式数据/HiAI Foundation/端侧 LLM） | 重写 |
| UI 层 | React + Tailwind，8 屏 | ArkTS/ArkUI 重写 | 0%（最大工作量） |
| 处理管线 | store 编排 | 逻辑复用，状态管理换 | 70% |
| 服务端 | Node 代理（LLM/STT/VLM/地理编码/配额） | 原样复用 | 100% |

**建议集成的 3+ 鸿蒙特性**（应用方向建议）：
1. **端侧 AI / HiAI Foundation**——LLM 分类/聚合走端侧，强化「本地优先隐私」故事（BYOK 之外的卖点）
2. **分布式数据管理**——替代 IndexedDB，手机记→平板/PC 看，多端同步
3. **原子化服务 / 元服务卡片**——桌面卡片快速记一笔，契合「随手记」
4. （可选）**AI 意图框架**——系统级识别「记一下…」意图唤起应用
