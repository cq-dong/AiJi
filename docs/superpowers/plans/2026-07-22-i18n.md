# i18n（中文 / English）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** App 全部 UI 文案中/英双语 + 运行时切换 + 首启选语言 + LLM 新输出跟随语言。

**Architecture:** 轻量自研 i18n——扁平点路径 key 字典按屏分片（`src/app/i18n/{zh,en}/<screen>.ts`），`t()` 非 hook 全局可调，`useT()` React hook 订阅 store 触发重渲；语言存 `Settings.language`，零依赖单例 `currentLang.ts` 供非 React 上下文（提示词/帮助函数）读取。设计细节见 `docs/superpowers/specs/2026-07-22-i18n-design.md`。

**Tech Stack:** React 19 + TS strict + Zustand + Vitest。零新依赖。

## Global Constraints

- `verbatimModuleSyntax` → 类型 import 必须 `import type { ... }`。
- `erasableSyntaxOnly` → 禁 enum/namespace/构造参数属性。
- `noUnusedLocals/Parameters` → 无未用变量。
- 子智能体自检用 `npx tsc -p tsconfig.app.json`，**禁** `npm run typecheck` / `tsc -b`（共享 tsbuildinfo 并发竞态）。
- 子智能体**只写自己任务列出的文件**，不碰他人屏目录、不碰 `zh/common.ts` / `en/common.ts` / barrel / core。shared 读自由。
- 子智能体**不 commit**，完成只回报；commit/push 由 lead 做。
- key 命名：屏前缀 `home.` / `chat.` / `common.` 等，全小写点路径（如 `home.empty.title`）。
- 字典文件**禁止**出现 `export const zh = {...}` 整体对象——每个片段文件只导出本屏片段（见 Task 1 模板）。
- 历史已生成数据（旧分类名/摘要）不翻译；seed 数据不翻译。

- [ ] **Step 3: 实现 currentLang.ts**

```ts
// 当前界面语言（i18n 键）。零依赖单例（防循环 import）：store ↔ adapters ↔ i18n 都依赖它，
// 它不依赖任何业务模块。与 currentOwner.ts 同构。
// 模块 init 同步 detect（onboarding 未 hydrate 时也有正确默认值，防首帧闪烁）。
export type Lang = 'zh' | 'en'

// 可注入便于测试：默认读 navigator.language。
export function detectLang(navLang?: string): Lang {
  const l = navLang ?? (typeof navigator !== 'undefined' ? navigator.language : 'zh')
  return l.toLowerCase().startsWith('zh') ? 'zh' : 'en'
}

let current: Lang = detectLang()

export function getCurrentLang(): Lang {
  return current
}

export function setCurrentLang(l: Lang): void {
  current = l
}
```

- [ ] **Step 5: 实现 `src/app/i18n/index.ts`（t）**

```ts
// t()：非 hook 全局翻译函数（helpers/适配器/提示词构建可用）。
// React 组件请用 useT()（./useT.ts）——本文件刻意不 import store，防 ESM 环
// （store → di → adapters → i18n → store）。
import { getCurrentLang } from '@/app/currentLang'
import { zh, type I18nKey } from './zh'
import { en } from './en'
import type { Lang } from '@/app/currentLang'

const dicts: Record<Lang, Record<string, string>> = { zh, en }

export type { I18nKey }

export function t(key: I18nKey, params?: Record<string, string | number>): string {
  let s = dicts[getCurrentLang()][key] ?? zh[key] ?? key
  if (params) for (const [k, v] of Object.entries(params)) s = s.replaceAll(`{${k}}`, String(v))
  return s
}
```

- [ ] **Step 7: 实现 `src/app/i18n/errorText.ts`**

```ts
// 服务器/适配器错误（Error('AUTH_409:…') 格式）→ 当前语言文案。
// 映射表在 common 片段的 error.* key；未命中回落原文（不丢信息）。
import { t } from './index'
import { zh, type I18nKey } from './zh'

export function localizeError(err: unknown): string {
  const msg = err instanceof Error ? err.message : String(err)
  const code = msg.split(':')[0]
  const key = `error.${code}`
  // 动态 key：类型收窄到 I18nKey 仅当 zh 里确实存在该 key
  if (key in zh) return t(key as I18nKey)
  return msg
}
```

### Task 4~12: 屏级文案抽取（9 个子智能体并行）

**统一作业规程（每个 agent 必读）：**

1. **只写**：自己负责的 `src/ui/screens/<x>/**`（或 components/layout）+ 自己的两个片段文件 `src/app/i18n/zh/<x>.ts`、`src/app/i18n/en/<x>.ts`。**禁碰** common 片段、barrel、core、他人屏目录、store、router。
2. **流程**：通读负责文件 → 找出所有用户可见中文（JSX 文本、placeholder、title/aria、helpers 里的标签、toast/alert、EmptyState、按钮）→ 起 key（`<屏>.` 前缀，语义命名如 `capture.hint.recording`）→ zh 片段写中文原文 → en 片段写自然地道的英文（不是逐字机翻；长度注意移动端 390px 不溢出，尽量不比中文长太多）→ 代码替换：
   - 组件内：`import { useT } from '@/app/i18n/useT'` → `const t = useT()` → `{t('home.empty.title')}`
   - 非组件函数（helpers.ts）：`import { t } from '@/app/i18n'` 直接调用（它在调用时读当前语言，切语言后下一次渲染自然正确）。
   - 带变量的字符串用 `{name}` 插值：`t('home.todayCount', { count: 3 })`。
   - 数组常量（如 onboarding 的 FEATURES）改为在组件内由 t() 构建。
3. **不改**任何逻辑/样式/className；不翻译 console 日志、注释、错误 throw 的内部消息（UI 直接展示给用户的除外——那种接到 localizeError 或起字典 key）。
4. **自检**：`npx tsc -p tsconfig.app.json` 绿（en 缺 key 会挂——类型强制）；`npx vitest run` 相关测试绿。grep 自查：`grep -n "[一-鿿]" src/ui/screens/<x>/**/*.tsx` 应只剩注释/日志。
5. **回报**：完成 + key 数量 + 遗留问题。不 commit。
